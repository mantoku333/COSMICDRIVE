 ## 🎮 作品概要
本作品はLive2D・Chedを用いた、非エンジニアでもコードを書き換える必要なく新規楽曲を実装できるリズムゲームです。

## ✨ セールスポイント

### 1曲当たりの実装コスト最小化

最も重視されるべきは1曲あたりの実装コストだという考え方を軸に、
コンテンツ追加のためにコードを書き換える必要のない運用を意識したシステムを構築。
エディタ上で行われる譜面制作側でスキル発動や楽曲終了通知などをコマンドとして埋め込みパース工程でそれらを読み取る設計により、**GUI操作のみで楽曲の実装**が可能に。

### Mbt方式による可変BPM対応

秒数ではなくMIDI規格の考え方に基づいたMbt方式でパースすることにより、
曲中のBPM変化を始めとする**イベントの視覚的な配置**を実現。
ロード時にテンポマップを前処理することで、**変拍子や可変BPM**に対応。

### 高密度地帯でも安定性を担保

Windowsの標準入力処理に依存しない**Raw Input**実装による入力遅延低減やマルチスレッド
ノーツのインスタンシング＋フラスタムカリングの思想を元にした**アクティブ状態管理**
レーンごとに走査位置を進め常にn個先のノーツとだけ比較処理を行う判定方式により探索範囲を局所化。高いスケーラビリティを確保することで、高密度の譜面でも安定性を担保。

| 操作 | キー |
|---|---|

| 設定画面の上下移動 | `W/S` または `↑/↓` |

> 🔧 キーコンは `Config` 画面から変更可能です。

## 📄 依存ライブラリ
このリポジトリ内に以下を同梱しています（`chumini/` 配下）。
- ImGui (`imgui/`)
- Live2D Cubism SDK (`Live2D/`)
- Effekseer (`Effekseer/`)
- DirectXTK ヘッダ群 (`directxtk/include`)

`chumini.vcxproj` は相対パス参照ベースの構成です。

## 🔍 実行時に必要なフォルダ
実行時は `chumini/` 直下の以下フォルダが必要です。
- `Assets/`
- `Songs/`
- `Save/`
- `Live2D/`
- `Effekseer/`
- `imgui/`

シェーダ類はビルド時に `Assets/Shader/*.cso` として出力されます。
